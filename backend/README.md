# GigCampus Backend

This backend uses Express, Supabase, and Zod to support user authentication and job posting/listing for the GigCampus platform.

## What it includes

- `Express` API server
- `Zod` validation for request payloads
- `Supabase` database access and auth integration
- Job posting with user ownership
- Filtering by category, location, is_remote, and pay rate
- Support for Supabase Realtime via the `jobs` table

## Install

1. Open a terminal in `backend/`
2. Run `npm install`

## Environment

Copy `.env.example` to `.env` and fill in your Supabase project values.

## Supabase setup

Create a `jobs` table in Supabase with the following fields:

- `id` (primary key)
- `title` (text)
- `description` (text)
- `pay_rate` (numeric)
- `category` (text)
- `deadline` (date)
- `is_remote` (boolean)
- `location` (text)
- `user_id` (uuid)
- `created_at` (timestamp with time zone)

### Example SQL

```sql
create table public.jobs (
  id bigint generated by default as identity primary key,
  title text not null,
  description text not null,
  pay_rate numeric not null,
  category text not null,
  deadline date not null,
  is_remote boolean not null,
  location text not null,
  user_id uuid not null,
  created_at timestamp with time zone default timezone('utc', now()) not null
);
```

## Run

- `npm run dev` — start the backend with hot reload
- `npm start` — start the backend in production mode

## API Endpoints

- `POST /api/auth/signup` — email/password signup
- `POST /api/auth/signin` — email/password sign in
- `POST /api/auth/otp` — send phone OTP sign in
- `GET /api/auth/oauth/google` — start Google OAuth redirect
- `GET /api/auth/user` — verify session token and return user info
- `GET /api/jobs` — list jobs with optional filters
- `POST /api/jobs` — create a new job (requires auth)

## Notes

- Use Supabase built-in auth in your frontend for session persistence.
- Frontend can subscribe to the `jobs` table with Supabase Realtime to receive new jobs automatically.
- Keep `SUPABASE_SERVICE_ROLE_KEY` secret; only use it on the backend.
